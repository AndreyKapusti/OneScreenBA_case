# Решение: solution.md 
## Выявление зависших остатков Wildberries (ФБО)

## Что сделано

Составлено техническое задание (ТЗ) для реализации отчета, который позволяет отслеживать товары, поступившие на склад Wildberries в рамках конкретных поставок, но не продающиеся длительное время.

**В ТЗ определены:**
- цель и назначение отчета;
- источники данных (API Wildberries) с указанием конкретных методов и полей;
- алгоритм сбора и связывания данных (прямая связка по `incomeID` = `supplyID`);
- формулы расчета показателей (принято, продано, возвраты, остаток, дни без продаж);
- макет итоговой таблицы с колонками;
- функциональные требования (фильтры, настройка порога дней без продаж).

**Ключевое техническое решение:** использование поля `incomeID` в отчете о продажах позволяет напрямую привязать каждую продажу к поставке.

---

## Что можно доработать при уточнении требований

При наличии возможности задать вопросы заказчику и уточнить детали эксплуатации, ТЗ можно дополнить следующими пунктами:

- **Период загрузки** — за какой срок загружать поставки: все исторические, ограничиться последними 3/6/12 месяцами или только теми поставками, где есть "зависшие" товары?
- **Сверка с API остатков** — нужна ли дополнительная сверка расчетного остатка с прямым API остатков для контроля расхождений? Уточнение: (метод API WB с остатками не подразумевает разделение товаров на поставки, но можно было бы вести суммарный баланс по всем поставкам для сравнения с данными от WB)
- **Скорость загрузки** - таблица с результатом формируется быстрее n секунд.
